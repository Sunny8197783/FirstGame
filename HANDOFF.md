## 1. 이 게임이 뭔가요?

**낮에는 전당포에서 물건을 감정·흥정하고, 밤에는 지하 격투장에서 베팅하거나 직접 싸우는 3막 21일 경영 시뮬레이션.** 단일 데모에서 시작해 정식 출시 품질로 끌어올리는 중입니다.

**핵심 재미 = "정보를 읽는 판단력 = 수익".** 방치하면 골드가 1G도 안 늘고, 모든 수익은 능동적 판단(흥정 밀당, 소문 추리, 텔 읽기)의 결과입니다.

### 하루 4박자 루프
| 시간 | 내용 |
|---|---|
| ☀️ 낮 | 손님이 **요구가**를 부르면 제시가로 밀당(인내심❤️·모욕 임계). 부위별 실물 검수(●1~5점)+구두 힌트(함정 섞임)로 진짜 가치 추정. 🎉진품(5%,×2)·🚨장물(압수) 변수. **진짜 가치는 저녁에 공개** |
| 🌆 저녁 | 매입품 되팔이 정산 — 낮의 판단이 여기서 채점됨 |
| 🌙 밤 | 파이터 베팅(하우스는 스탯·전적만 앎, **소문은 나만 아는 정보**). 소문 6테마 겹침=사실/충돌=거짓 추리. 경기 아크(압승/접전/역전+각성). 2경기 후 **직접 출전**(러시>견제>카운터 심리전, 텔 60%진짜/40%페인트) |
| 🌌 새벽 | 암시장: 빚 상환/대출, 도구 7종, 파이터 후원, (시즌 모드)가게 이전. **수동 저장** |

### 막마다 다른 '판' — 반복감 해소 장치
같은 4박자 루프가 21일 반복되지 않도록, 막마다 **고유 결정 구조**가 하나씩 얹힌다.

| 막 | 고유 시스템 | 결정 구조 |
|---|---|---|
| **1막** | (없음 — 생존/튜토리얼) | 밀당 흥정: 여러 라운드 협상 |
| **2막** | 🏪 **라이벌 전당포 '황금손'** | **밀봉 입찰 1회 승부** — 손님 40%가 대상. 라이벌 입찰가는 끝까지 비공개, 성향 힌트(`RIVAL_TELLS`)로만 추정. 낮으면 뺏기고 높으면 승자의 저주 |
| **3막** | 🔨 **경매의 날** (16·18·20일) | **공개 호가 경쟁** — 저녁과 밤 사이에 열림. 회장·나사장·수집가와 호가를 주고받으며 언제 손을 뗄지 결정. 상한선 비공개, 물러나는 순간이 곧 정보 |

세 시스템 모두 **정보 비대칭 원칙 유지**(상대의 숫자는 절대 노출 안 됨)이며, 흥정 엔진은 건드리지 않은 **별도 경로**다. 매입 처리는 `closeDeal`/`S.purchases`를 재사용해 통계·업적이 어긋나지 않는다.

### ⚠️ 밸런스 튜닝 시 주의 — 경제와 엔딩 임계는 한 몸이다
`CONFIG`의 경제 축(매물 가치·`ACT_CUSTOMER_BONUS` 손님 수·수익원)을 건드리면 **`ENDING_TRUE`/`ENDING_RING`/`FINAL_GRADES`도 반드시 같이 재보정**해야 한다. 실제로 2막 매물·물량을 올렸더니 고수 진엔딩이 42%→88%로 튀어 임계를 22만→35만으로 다시 올렸다.

- 판독법: `npm run balance` → **엔딩 도달률이 한쪽으로 쏠리면 임계가 경제와 어긋난 것.**
- 목표 분포: 방치=배드 100% / 무작위=대부분 배드 / 보통=5종 고루 / 고수=진엔딩 약 40~50%(독식 아님)

### 3막 스토리 + 시즌 모드
- **1막(1~7일)** 김사장의 빚(이자 5%/일 복리) → 8일 아침 강제 회수, 못 갚으면 배드엔딩
- **2막(8~14일)** 회장의 시험(순자산 40,000G) → 실패 시 엔딩
- **3막(15~21일)** 21일 밤 👑그랜드 파이널 → 순자산별 엔딩 3종
- **🌆 시즌 모드**(클리어 후 무한): 6주 시즌, 파이터 물갈이, 전설 세트 로테이션, 🏚️가게 이전 프레스티지

## 3. 시작하는 법

```bash
npm install
npm run dev        # 개발 서버 http://localhost:5173
npm run build      # 정적 빌드 → dist/
npm run simulate   # 회귀 시뮬(흥정·스파링) — 변경 후 반드시 통과 확인
npm run balance    # 흥정 밸런스 리포트(4가지 플레이 스타일)
```

게임 실행 후 **[D]키 = 디버그 패널**(진짜 가치·최저 수락가·소문 진위·EV 표시), HUD 우측 📈📊🏆⚙️ 아이콘으로 통계/확률/업적/설정 열람.

### ⚠️ 절대 원칙 — 보존 대상 (로직 변경 금지)
데모에서 검증된 시스템은 **동작을 바꾸면 개선이 아니라 버그**입니다:
- 3막 캠페인·엔딩 분기 / 밀당 흥정(INSULT_RATIO·ACCEPT_GAP) / 감정 정보구조(검수·함정·잭팟·장물)
- 소문 추리(겹침=사실/충돌=거짓) / 배당 공식 `(1/추정승률)×0.95` / 정보 비대칭(하우스는 소문 모름)
- 경기 아크·각성 / 스파링 심리전(텔 60/40) / 새벽 암시장 / **방치 보상=0**

- 신규/수정 코드는 `[Phase2]`, `[Phase3]` 주석으로 마킹 (이식 원문과 구분)
- **변경 후 `npm run simulate` 통과 필수** (흥정 매입가율·결렬률·스파링 승률이 기준 범위 유지되는지 회귀 검증)

## 8. Phase 5 — 시뮬레이션 인바리언트 (완료 · 커밋 `b6d145a`)

`sim/bots.mjs`가 **헤드리스 하루-루프 드라이버 + 봇 3종**을 제공한다(UI 연출은 건너뛰고 엔진만 구동, 승자 결정은 `runFight()` 원문 규칙 이식, 8·15일차 막 관문은 `startDay()` 규칙 반영).

| 봇 | 정책 | 검증하는 원칙 |
|---|---|---|
| `BOT_IDLE` | 아무것도 안 함 | **방치 보상 = 0** (21일 후 골드 변화 0G, 빚만 불어남) |
| `BOT_RANDOM` | 무작위 제시·베팅 | 하우스 마진에 마모돼 기대값 마이너스 |
| `BOT_SMART` | 최저 수락가·소문 진위 활용 | **판단력 = 수익** (흑자·진엔딩) |

- `npm run simulate` → haggle·challenge·**invariants** 3종 회귀 (변경 후 필수)
- `npm run balance` → 흥정 4스타일 리포트 + **하루 루프 리포트**(순자산 곡선·막 관문 통과율·엔딩 도달률·수익 출처)
- 밸런스 판독 기준: **방치 0 → 무작위 마모 → 정보 활용만 우상향**. 이 순서가 깨지면 밸런스가 무너진 것.
- 브라우저 전용 API(rAF 등)는 `src/ui/hud.js`처럼 코드에서 가드하거나 `sim/stub.mjs`에 추가한다.
